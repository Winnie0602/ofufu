# PRD v0.1: 台灣人學日文內容學習平台

## 文件資訊

| 項目 | 內容 |
| --- | --- |
| 版本 | v0.1 |
| 狀態 | Draft |
| 目標市場 | 台灣 |
| 主要使用者 | 繁體中文母語的日文學習者 |
| 目標程度 | JLPT N5-N3 |

## 產品定位

本專案將從以歌曲、歌詞與 YouTube 為核心的學習網站，逐步轉型為「台灣人學日文的內容學習平台」。

新產品以教材內容與學習行為為核心，歌曲仍會保留，但只是教材類型之一，不再決定整個系統的資料模型與播放架構。

核心學習流程：

```text
內容學習
-> 逐句理解
-> 點擊單字或文法
-> 收藏
-> 測驗
-> 複習
```

## 問題定義

目前產品以 `Song -> Lyric -> YouTube` 為核心，造成以下限制：

- Article、Dialogue 等非歌曲教材難以自然加入。
- 教材 ID、播放狀態與 YouTube Video ID 高度耦合。
- 測驗依賴歌詞與時間軸，難以重用於一般句子。
- 全站播放器讓不需要影音的頁面也依賴 YouTube。
- 付費價值若建立在第三方影片或歌詞全文上，商業化風險較高。

## MVP 目標

第一階段不建立完整語言平台，只驗證一條可重複使用的學習閉環：

```text
Article / Dialogue
-> Segment 逐句學習
-> 單字收藏
-> TTS 音檔
-> 題庫測驗
-> 學習紀錄
```

### 成功條件

- 使用者可以完成一篇 Article 或一組 Dialogue。
- 使用者可以查看逐句翻譯並播放日文音檔。
- 使用者可以從 Segment 查看並收藏 Vocabulary。
- 使用者可以完成由已發布題庫組成的測驗。
- 系統可以保存使用者的收藏與測驗紀錄。
- 新學習流程不依賴全站 YouTube 播放器。

## 目標使用者

### 主要 Persona

- 居住於台灣或以繁體中文為主要閱讀語言。
- 日文程度約 JLPT N5-N3。
- 希望透過真實、短篇且可理解的內容累積日文能力。
- 需要繁中翻譯、單字提示、音檔與低摩擦測驗。

## MVP 功能範圍

### Article

日文文章教材，支援：

- 文章列表與文章詳情。
- 逐句 Segment 顯示。
- 顯示或隱藏繁體中文翻譯。
- 播放 Segment TTS。
- 查看句中單字。
- 收藏單字。

### Dialogue

日文對話教材。MVP 統一使用兩位固定角色，以 A、B 你一句我一句的形式呈現。兩位角色的名字、頭像與 TTS 聲音由應用程式固定設定，不由每份教材自行建立。

支援：

- 依固定角色 A、B 顯示名字與頭像。
- 每次角色發言以一個 Dialogue Turn Segment 表示。
- 使用角色對應的固定 TTS 聲音。
- 逐句顯示或隱藏繁中翻譯。
- 播放每個 Dialogue Turn 的 TTS。
- 查看句中單字。
- 收藏單字。

MVP 不支援自訂角色、三人以上對話或每篇教材使用不同角色設定。

### Vocabulary

單字資料支援：

- 單字原形與表面形式。
- 假名。
- 繁體中文意思。
- 詞性。
- 例句。
- 使用者收藏。
- 從原始 Material 或 Segment 返回內容脈絡。

### Quiz

第一版題型：

- 單字選擇題。
- 聽音選擇題。
- 漢字選假名。
- 句子翻譯選擇題。

正式題目來自 Question Bank，不在使用者作答時隨機產生低品質干擾項。

題目內容流程：

```text
AI 或管理者產生候選題
-> 管理者審核
-> 發布至 Question Bank
-> 正式測驗使用
```

### Membership

MVP 的 Membership 僅包含：

- 使用者登入。
- 使用者資料識別與權限。
- 單字收藏的資料歸屬。
- 測驗與學習紀錄的資料歸屬。

第一階段不包含訂閱、付款或方案管理。

## 歌曲系統定位

歌曲保留為 `Material Type = song`，但不再作為底層核心。

### 保留

- Song 頁。
- Song 頁內的 YouTube Embed。
- 歌詞顯示。
- 翻譯顯示。
- 未來可加入歌詞單字標記、文法標記與收藏。

### 移除

- 全站背景 YouTube 播放器。
- 跨頁持續播放。
- 非 Song 頁對 YouTube player state 的依賴。

### 原因

- 降低 YouTube 政策與第三方平台依賴風險。
- 降低商業化對第三方內容的依賴。
- 讓 Article、Dialogue、Vocabulary 與 Quiz 能獨立運作。
- 將付費價值建立在學習服務，而非影片播放本身。

## Audio 方向

TTS 音檔不存入 MongoDB。系統使用以下流程：

```text
解析 Audio Asset
-> Cache miss 時產生 MP3
-> 上傳 Cloudflare R2
-> MongoDB 保存 storage key 與資產狀態
-> 後續播放使用快取音檔
```

正式教材應優先在發布時預先產生音檔；使用者第一次播放時的 lazy generation 僅作為補救流程。

## 非目標

第一階段不做：

- 英文學習。
- 訂閱付款。
- 聊天室、留言或社群。
- 完整 SRS。
- 完整 Grammar domain。
- 全站背景 YouTube 播放器。
- AI 內容未經審核直接發布。
- 多國語教材與四種 UI locale 的同步擴充。

## 產品原則

- **學習優先：** 每個功能都應服務於理解、收藏、測驗或複習。
- **內容可追溯：** Vocabulary、Annotation 與 Question 應能回到來源 Segment。
- **AI 可審核：** AI 只建立候選內容，不直接成為正式教材。
- **第三方可替換：** YouTube、TTS 與儲存服務不得成為 domain primary key。
- **聚焦市場：** MVP 只正式支援 `zh-TW -> ja`。

## 風險與待驗證事項

- Article 與 Dialogue 是否能帶來足夠的回訪動機。
- 使用者是否真的會從教材收藏單字並回來複習。
- 四種 Quiz 題型是否超出第一版的內容製作能力。
- 人工審核 Question Bank 的成本是否可持續。
- TTS、AI 與物件儲存成本是否符合未來商業模型。
- 歌詞、翻譯、新聞或其他第三方內容的授權範圍需另行確認。

## 建議衡量指標

- Material 開始率與完成率。
- 翻譯顯示率與 TTS 播放率。
- 每個完成 Material 的單字收藏數。
- 收藏後七日內的複習率。
- Quiz 開始率、完成率與答題正確率。
- Question 被管理者退回或修改的比例。
